This is a Claude Code plugin (for Anthropic's Claude Code product), not a Cortex Code plugin. These are two different products with different plugin/extension systems, so this plugin cannot be directly installed in Cortex Code.

However, the core concept ("Ralph Loop") is straightforward and can be replicated in Cortex Code. Here's what it does and how you can achieve the same thing:

What Ralph Loop Does

It's an iterative feedback loop: you give the agent a task, and instead of exiting when it thinks it's done, a hook intercepts the exit and re-feeds the same prompt, forcing the agent to keep iterating until a completion condition is met. This is useful for tasks like "get all tests passing" where multiple iterations of code-fix-test cycles are needed.

How to Achieve This in Cortex Code

Cortex Code doesn't have a Claude Code plugin system, but you have two practical options:

Option 1: External bash loop (simplest)

while true; do
  cortex -p "Your task prompt here. When done, create a file called DONE.txt" --allowedTools '*'
  if [ -f DONE.txt ]; then
    rm DONE.txt
    echo "Task complete!"
    break
  fi
done

Option 2: Use Cortex Code's hooks system

Cortex Code supports hooks that run on events. You can configure a hook in your project's .cortex/settings.json or ~/.snowflake/cortex/settings.json:

{
  "hooks": {
    "stop": [
      {
        "command": "bash /path/to/your/stop-hook.sh",
        "description": "Ralph-style loop hook"
      }
    ]
  }
}

The stop hook script would check if the task is complete and block the exit if not.

Option 3: Just ask Cortex Code to iterate

In practice, you can often just prompt Cortex Code within a session to keep going:

"Implement X following TDD. Run tests after each change. Keep fixing until all tests pass. Do not stop until every test is green."

Cortex Code will iteratively work through the problem within a single session without needing an external loop.

Bottom line: The ralph-loop plugin is designed for Claude Code's specific plugin architecture and won't install in Cortex Code. But the technique itself (iterative agent loops) works fine in Cortex Code using a bash wrapper or hooks.

No, Cortex Code does not have a "plugin" system like Claude Code. They are different products with different extension architectures. But Cortex Code has three extensibility mechanisms that can replicate what most Claude Code plugins do:

1. Skills (closest equivalent to plugins)

Skills are reusable instruction sets. You can create custom ones or load them from remote repos.

Create a project-level skill:

mkdir -p .cortex/skills/my-skill/
# Create .cortex/skills/my-skill/SKILL.md

Create a global skill:

mkdir -p ~/.snowflake/cortex/skills/my-skill/
# Create ~/.snowflake/cortex/skills/my-skill/SKILL.md

Load remote skills (e.g., from a GitHub repo) via ~/.snowflake/cortex/skills.json:

{
  "remote": [
    {
      "source": "https://github.com/org/skills-repo",
      "ref": "main",
      "skills": [{ "name": "skill-name" }]
    }
  ]
}

Invoke with $skill-name in the prompt.

2. Hooks (for lifecycle events like the Ralph Loop stop hook)

Configured in ~/.snowflake/cortex/hooks.json. Supports events like Stop, PreToolUse, PostToolUse, SessionStart, etc.

{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          { "type": "command", "command": "/path/to/stop-hook.sh" }
        ]
      }
    ]
  }
}

3. MCP Servers (for external tool integrations)

Configured in ~/.snowflake/cortex/mcp.json:

{
  "mcpServers": {
    "my-server": {
      "command": "npx",
      "args": ["-y", "@scope/mcp-server"]
    }
  }
}

So for that Ralph Loop plugin specifically:

The plugin has three components -- a slash command, a stop hook, and scripts. You'd need to manually port it:

  1. Stop hook -> goes into hooks.json under the Stop event
  2. Commands -> create as a Cortex Code skill (in .cortex/skills/ralph-loop/SKILL.md)
  3. Scripts -> keep as bash scripts referenced by the hook/skill

There's no cortex plugin install command that would auto-install a Claude Code plugin. The formats are different and not compatible. But the building blocks (skills + hooks + MCP) cover the same ground -- it just requires manual adaptation.

That skill combines three actions into one flow:

  1. Update docs (Step 3 in the skill) -- automatically updates files in your docs/ folder:
  2. docs/changelog.md -- adds entries for new features/fixes
  3. docs/architecture.md -- only if structural changes occurred

  4. docs/project_status.md -- moves completed items, updates progress

  5. Commit -- stages changes, generates a commit message, and commits

  6. Push + PR -- pushes to remote and creates a pull request via gh pr create

How to use it

You can invoke it by typing:

$commit-commands create a PR

Or more specifically, use the /commit-push-pr command pattern described in the skill. The skill routes based on your intent:

Request Route
"commit my changes" /commit only
"create a PR" / "push and create PR" /commit-push-pr (full flow)
"clean up branches" /clean-gone
"update docs" Triggers doc updates

It's a global skill you've installed at ~/.snowflake/cortex/skills/commit-commands/, so it's available in any project.
