Ralph Wiggum Loop

Implementation of the Ralph Wiggum technique for iterative, self-referential AI development loops.

What is Ralph Loop?

A self-referential feedback loop where the same prompt is fed back repeatedly. Each iteration, you see your previous work in files and git history, allowing incremental improvement until a task is done.

Commands

Starting a loop

When the user says $ralph-loop or asks to start a ralph loop, run the setup script:

bash /Users/mcastro/Documents/github/sfc-gh-mcastro/1-cortex-code-cli/skills/ralph-loop/scripts/setup-ralph-loop.sh <PROMPT> [OPTIONS]

Options: - --max-iterations <n> - Max iterations before auto-stop (default: unlimited) - --completion-promise <text> - Phrase that signals completion

Example:

bash /Users/mcastro/Documents/github/sfc-gh-mcastro/1-cortex-code-cli/skills/ralph-loop/scripts/setup-ralph-loop.sh "Build a REST API for todos" --completion-promise "DONE" --max-iterations 20

After running the script, immediately begin working on the task described in the prompt.

Cancelling a loop

When the user says $cancel-ralph or asks to cancel the ralph loop:

  1. Check if .cortex/ralph-loop.local.md exists: bash test -f .cortex/ralph-loop.local.md && echo "EXISTS" || echo "NOT_FOUND"

  2. If NOT_FOUND: Say "No active Ralph loop found."

  3. If EXISTS:

  4. Read .cortex/ralph-loop.local.md to get the current iteration number
  5. Remove the file: rm .cortex/ralph-loop.local.md
  6. Report: "Cancelled Ralph loop (was at iteration N)"

Completion Promise Rules

CRITICAL: If a completion promise is set, you may ONLY output the promise tag when the statement is completely and unequivocally TRUE.

  • Use <promise>YOUR_PHRASE</promise> XML tags exactly
  • Do NOT output false promises to escape the loop
  • Do NOT lie even if you think you should exit
  • The loop is designed to continue until genuine completion

When to Use

Good for: - Well-defined tasks with clear success criteria - Tasks requiring iteration and refinement (getting tests to pass) - Greenfield projects where you can walk away - Tasks with automatic verification (tests, linters)

Not good for: - Tasks requiring human judgment or design decisions - One-shot operations - Tasks with unclear success criteria

What is Ralph Loop?

A self-referential feedback loop where the same prompt is fed back repeatedly. Each iteration, you see your previous work in files and git history, allowing incremental improvement until a task is done.
